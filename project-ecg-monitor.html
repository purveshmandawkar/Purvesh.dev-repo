IoT ECG & Heart Rate Monitor

A portable, low-cost ECG system that captures the heart's electrical activity from skin-surface electrodes, conditions the microvolt-level signal through an analog front-end, and streams real-time heart rate to a mobile dashboard over Wi-Fi.

Analog Signal Chain IoT Platform Edge Processing Biopotential Sensing Embedded C

What It Does

Unlike bulky clinical ECG machines, this prototype demonstrates how low-cost hardware can monitor cardiac health remotely. The system captures electrical bio-signals from the body, filters out noise, detects heartbeats in real time, and displays live BPM on a local OLED display and a remote mobile app simultaneously.

Signal-to-Cloud Pipeline

01 CAPTURE

Electrode Placement

Three Ag/AgCl adhesive electrodes placed in Einthoven's Triangle configuration pick up biopotential signals from the skin surface. These signals are in the microvolt range — smaller than the noise from muscle movement or power line interference.

02 CONDITION

Analog Front-End (AD8232)

The AD8232 module amplifies the signal and applies a band-pass filter. This removes power line noise (50Hz) and muscle tremor artifacts, leaving only the cardiac frequency range for processing.

03 PROCESS

R-Peak Detection

The IoT board's 12-bit ADC converts the analog signal. A peak-detection algorithm identifies the R-peak of the QRS complex — the sharp spike in each heartbeat — and calculates the RR interval to derive BPM.

04 OUTPUT

Dual Display

Calculated BPM is displayed locally on a 0.96" OLED and simultaneously pushed to Blynk Cloud over Wi-Fi for remote mobile monitoring from any location.

Source Code

ECG Monitor Firmware — Leads-off detection + Blynk streaming

#define BLYNK_TEMPLATE_ID "YOUR_TEMPLATE_ID"
#define BLYNK_DEVICE_NAME "ECG_Monitor"

#include <WiFi.h>
#include <BlynkSimpleEsp32.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

#define SENSOR_PIN   34
#define LO_PLUS      32   // Leads-off detection +
#define LO_MINUS     35   // Leads-off detection -

Adafruit_SSD1306 display(128, 64, &Wire, -1);

char auth[] = "Your_Blynk_Auth_Token";
char ssid[] = "Your_WiFi_SSID";
char pass[] = "Your_WiFi_Password";

void setup() {
  display.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  display.clearDisplay();

  pinMode(LO_PLUS,  INPUT);
  pinMode(LO_MINUS, INPUT);

  Blynk.begin(auth, ssid, pass);
}

void loop() {
  Blynk.run();

  // Check if electrodes are disconnected
  if (digitalRead(LO_PLUS) || digitalRead(LO_MINUS)) {
    display.clearDisplay();
    display.setCursor(0, 10);
    display.println("LEADS DISCONNECTED");
    display.display();
  } else {
    int signal = analogRead(SENSOR_PIN);

    display.clearDisplay();
    display.setCursor(0, 0);
    display.print("ECG Monitoring...");
    display.setCursor(0, 30);
    display.print("Signal: ");
    display.print(signal);
    display.display();

    Blynk.virtualWrite(V1, signal);
  }

  delay(10);  // ~100Hz sampling rate
}

Key Insight

"Stable filtering is everything when dealing with microvolt-level bio-signals. The AD8232 does the heavy lifting — without proper analog conditioning, all you capture is noise from muscle movement and 50Hz power lines."
