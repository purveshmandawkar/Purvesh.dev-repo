Fail-Safe 220V AC Pump Controller

An industrial-grade control system for a 1HP municipal water pump. Built to handle the three real-world failure modes that destroy standard relay setups — inrush surge, water turbulence, and microcontroller freeze.

Power Electronics Real-time Firmware Watchdog Timer Relay Control Fail-safe Design Embedded C

The Problem

Standard smart plugs and basic Arduino relays fail on heavy industrial loads. Automating a 1HP water pump introduces three engineering challenges that most hobbyist builds completely ignore:

Problem 1 — Inrush Surge

A 1HP motor pulls a massive current spike on startup. When it shuts off, it shoots a high-voltage spike backward (back-EMF). This can weld standard relay contacts shut permanently.

Problem 2 — Water Turbulence

Water entering a tank creates waves. These waves cause the float switch to bounce rapidly — forcing the pump to violently switch ON and OFF like a machine gun. This destroys the motor and the relay within days.

Problem 3 — Microcontroller Freeze

Power fluctuations in a real environment can freeze the microcontroller. If it freezes while the pump is running, the tank runs dry and the motor burns out. No standard relay project handles this.

The Solution — Multi-layered Architecture

The system is divided into three strictly isolated zones to prevent high-voltage arcing from reaching the control logic:

Layer 1 — Power Isolation

HLK-PM01 step-down converter safely converts 220V AC to clean 5V DC. A 2A glass fuse and MOV (Metal Oxide Varistor) protect against grid voltage spikes before they reach the electronics.

Layer 2 — Control Logic (3.3V)

IoT development board runs all logic on a safe, isolated 3.3V signal. Float switch input is stabilized with a 10kΩ pull-up resistor to prevent false triggers from electromagnetic noise.

Layer 3 — High-power Actuation (220V)

30A mechanical relay handles the 20A motor load. An RC snubber circuit wired across the relay contacts absorbs the destructive back-EMF spike every time the motor shuts off. A parallel 16A manual bypass switch keeps the pump running if the digital brain is destroyed by a lightning strike.

Firmware Architecture

Standard hobbyist code uses delay(), which completely freezes the processor. This firmware uses a non-blocking state machine so the processor is always awake and monitoring safety protocols.

Two key software features solve the turbulence and freeze problems:

Source Code — Non-blocking State Machine with Watchdog

#include <esp_task_wdt.h>

// --- SYSTEM CONFIGURATION ---
#define WDT_TIMEOUT     3       // Watchdog panic threshold (seconds)
const int FLOAT_PIN   = 18;    // Tank sensor input
const int RELAY_PIN   = 19;    // 30A Relay trigger
const unsigned long DEBOUNCE_DELAY = 5000; // 5-sec wave turbulence filter

// --- STATE MEMORY ---
bool isPumpRunning        = false;
int lastSteadyState       = HIGH;
int lastRawState          = HIGH;
unsigned long debounceTime = 0;

void setup() {
  // Configure relay FIRST (safety priority)
  pinMode(RELAY_PIN, OUTPUT);
  digitalWrite(RELAY_PIN, LOW);  // Force OFF on boot

  // Configure sensor with internal pull-up
  pinMode(FLOAT_PIN, INPUT_PULLUP);

  // Arm the hardware watchdog
  esp_task_wdt_init(WDT_TIMEOUT, true);
  esp_task_wdt_add(NULL);
}

void loop() {
  // PHASE 1: SURVIVAL — kick the watchdog (prevents freeze lockup)
  esp_task_wdt_reset();

  // PHASE 2: SENSING
  int currentRaw = digitalRead(FLOAT_PIN);

  // PHASE 3: DEBOUNCE — ignore wave turbulence
  if (currentRaw != lastRawState) {
    debounceTime = millis();  // Reset timer on any change
  }

  // Only act if sensor stable for 5 full seconds
  if ((millis() - debounceTime) > DEBOUNCE_DELAY) {
    if (currentRaw != lastSteadyState) {
      lastSteadyState = currentRaw;

      if (lastSteadyState == LOW && !isPumpRunning) {
        digitalWrite(RELAY_PIN, HIGH);  // Water arrived: Pump ON
        isPumpRunning = true;
      }
      else if (lastSteadyState == HIGH && isPumpRunning) {
        digitalWrite(RELAY_PIN, LOW);   // Water dropped: Pump OFF (dry-run protection)
        isPumpRunning = false;
      }
    }
  }

  lastRawState = currentRaw;
}

Key Engineering Decisions

DecisionWhy
RC Snubber across relayWithout it, the back-EMF spike from the inductive motor will eventually weld relay contacts shut
5-second debounce filterWater takes 3–5 seconds to settle after the pump changes state — any shorter causes chatter
Hardware watchdog (3s)If firmware ever freezes mid-cycle, system reboots and forces pump to safe OFF state
Active-low relay initRelay starts OFF on power-up — safe default prevents pump from running on boot
Manual bypass switchIf the entire board is destroyed by lightning, water supply is not cut off

Key Insight

"Real-world control systems fail from physics, not just code — snubbers, debounce, and watchdogs are not optional. They're the difference between a project that survives a week and one that runs for years."
